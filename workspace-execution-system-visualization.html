01

The core operating loop

Every feature exists to keep this loop turning. An external (IDT) or internal (BDT) signal becomes a node, then a card you can reason with, then a decision/task/project, executed and measured back into the twin — which produces the next recommendation.

Signal→ Node→ Card→ Discussion→ Decision / Task / Project→ Execution→ Metric impact→ Twin update↺ New recommendation

⤴Import / “Export to Workspace”

🔗Multi-card connections

Where: center canvas
04

Workspace modes

Modes replace today’s confusing 0→100 expand-scroll. The mode decides which surfaces show, instead of continuously resizing the whole grid. Top mode bar

Explore
Decision
Execution
Review
Agent
Explore — gather and understand context by importing BDT/IDT nodes as cards. Best for: founder, strategy, product, sales, investor. Canvas shows imported cards + live signals.
Decision — compare cards side-by-side, simulate options, choose a path (ghost outcome objects). Best for: founders, department heads, project owners.
Execution — convert cards into tasks/projects and track work; hero collapses, kanban & project board take focus. Best for: project teams, members, managers.
Review — measure what changed: estimated vs actual metric impact, goal variance, what must happen next. Best for: leadership, departments, investors, board.
Agent — delegate implementation steps to agents with a context pack, plan approval and reviewable outputs. Best for: members, operators, builders, analysts.
05

Project management spaces

The PM layer is not a separate tool — it lives under the Projects item in the left rail and is represented as nodes back in the BDT. Three nested spaces serve three altitudes.

🛰 Superspace

Founders · super-admin · CXOs · dept heads
  • All active products & projects
  • Project health, delays, blockers, budget
  • Goal-alignment & metric-impact map
  • Resource load & over-allocation
  • Decision queue (approve / reject)
  • Create projects · assign members

📁 Project / Product Space

Project owner · dept heads · assigned members
  • Project context & source cards
  • Goal links (daily → long-term)
  • Milestones & dependencies
  • Tasks · decisions · risks
  • Files / evidence · project chat
  • Configured agents · metric impact

👤 Member Space

Each individual contributor
  • Unified task inbox (all projects)
  • “Today” plan by deadline + impact
  • Goal context per task
  • Task chat & agent execution
  • Dependency alerts
  • Personal metrics (velocity, quality)

RBAC note: company-level roles already exist; we add a project-level tier (owner / manager / contributor / viewer) so managers can create projects and assign multiple members.

12

MVP build order

Prove the core loop first: node → card → chat → task/project → goal link → metric impact → update.

Phase 1 · Card Workspace

Import nodes as cards, card chat, manual updates, links. Reason with multiple cards, convert one to a task.

Phase 2 · Project Spaces

Superspace, project & member spaces with tasks + ownership.

Phase 3 · Goal Alignment

Daily→annual hierarchy and goal links on every project/task.

Phase 4 · Metric Impact

Dept/company metric model, estimated vs actual tracking.

Phase 5 · Suggestive Tasks

Recommendations from gaps, goals, blocked dependencies.

Phase 6 · Agents

Attach agents to card/task/project with review & approval.
