Pre-req — open the workspace
Saved Nodes (IDT → Canvas)
1 · Save a node → it appears on the canvas Done
  1. On 3D Twin, click into a company planet and open an action node.
  2. Click Save Workflow (top-right). Button flips to Saved.
  3. Close it, then click Open Workspace → left rail Canvas (default).
Expect: the node shows as a card under Saved Nodes with role chip, breadcrumb, and time.
2 · Export to Workspace (from the IDT) Done
  1. Open a fresh action node from the universe.
  2. Click Export To Workspace (top-right).
Expect: the workspace opens focused on that node's card.
Also verify the fix: open a node that's already saved from the Canvas → the Export button is now hidden (was dead before).
3 · Open node inline (ActionNodeWorkspace overlay) Done
  1. Click a saved card → its InfoCard opens.
  2. If it's an action-level node, click Open Workspace.
Expect: the full node workspace scales in smoothly as a full-cover overlay. Close Workspace returns you to the card.
4 · InfoCard — chat, tabs, Link to Goal Partial
  1. Open a saved card. Use Card Chat (right): try Summarize / Why it matters / Generate tasks. Toggle Voice (visual only — no backend token yet).
  2. Browse the tabs: Overview · Relationships · Evidence · History.
  3. Click Link to Goal → toast confirms.
Expect: chat replies with node-aware text; Link to Goal produces a real goal entry.
Known partial: metric tiles, relationships, evidence, version history are derived placeholders. Compare / Assign / Agent / Simulate show "later phase."
5 · Card → Add to PM Project New Done
  1. Open any saved node's InfoCard.
  2. Click Add to Project (violet button).
  3. In the modal, choose a project (e.g. "Canvas v2 Launch") → click Add Task.
  4. Navigate to Projects → Superspace → open that project → Tasks tab.
Expect: a new task "[Node title] — follow up" appears in the To Do column of the selected project's kanban.
6 · Card → Convert to Project New Done
  1. Open any saved node's InfoCard.
  2. Click Convert to Project (indigo button) → confirm in the modal.
  3. Navigate to Projects → Superspace.
Expect: a new project card appears in Superspace with the node's title as project name and its description pre-filled. You are set as owner.

Project / Product Space (depth)
12 · Project Space — tasks, milestones, decisions, risks, files, chat Done
  1. Open Canvas v2 Launch. Check the sub-tab bar: Tasks · Milestones · Decisions · Risks · Files · Chat.
  2. Tasks: add a task (name + assignee) → it lands in To Do. Drag it to Done.
  3. Milestones: click + Milestone → add title + date → tick the checkbox to complete it.
  4. Decisions: click + Decision → fill title + rationale → approve/reject an existing open one.
  5. Risks: add a high-severity risk → toggle it to Mitigated. Verify the health score on Superspace updates.
  6. Files: add a doc link.
  7. Chat: type a message → click Ask AI.
Expect: every sub-tab works independently, persists to localStorage, and survives a page reload. The AI chat reply references real project data (tasks, risks, decisions).

  1. Projects → My Work tab.

  3. In All Tasks view, drag a task to Done → switch back to Today Plan.
Expect: "Done this week" increments by 1. "Overdue" count is correct (tasks past their due date). On-time rate is a live percentage.
15 · Today Plan — sorted by deadline + goal context New Done
  1. Projects → My Work → confirm you're on the Today Plan tab (default).
  2. Verify tasks are sorted: tasks with the soonest due date first; tasks without a due date appear last.
  3. Look for the goal chip (🎯 icon) under each task — it should name the linked goal (e.g. "Launch Canvas v2").
  4. Tick the circle button on a task to mark it done in place.
  5. Switch to All Tasks tab → verify the full kanban across all your projects is still there.
Expect: overdue tasks show a red "Overdue ·" prefix. In-progress tasks with the same due date bubble above todo tasks. Checking a task marks it done immediately (no reload needed).
16 · My Work all-tasks kanban — status change syncs back to project Done
  1. My Work → All Tasks tab.
  2. Drag a task from To Do → Done.
  3. Navigate to the owning project in Superspace and open it.
Expect: the same task's status is Done in the project kanban (single source of truth).
Goals & Metrics Superalignment
